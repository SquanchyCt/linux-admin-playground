### Среда
Docker Engine  
Ubuntu Server

### Начало

Запускаем команду:  
docker run -dit --name UbuntuServer ubuntu:24.04 - запуск контейнера с образом ubuntu:24.04  
docker exec -it UbuntuServer bash - открытие контейнера в интерактивном терминале c bash консолью
apt update && apt upgrade -y - обновим системные пакеты и сразу же их установим

### Создание файловой системы

1. Прежде чем начать создавать каталоги, посмотрим где мы находимся:  
    ls - команда показывающая какие каталоги доступны для открытия    
2. Создание каталогов:  
    cd /opt - заходим в каталог /opt  
    mkdir devops-app - создаем каталог devops-app  
    cd .. - выходим на директорию ниже, отсюда будем создавать все остальные директории прописывая полный путь  
    mkdir /logs  
    cp -r /logs /backups - копируем каталог /logs(с флагом рекурсии) и создаем на его основе новый каталог с именем /backups  
    cp -r /logs /scripts  
3. Поиск файлов .log и текста по всем файлам  
    Для примера попробуем найти все файлы ".log" больше 10МВ:  
    find . -name ".log" -size +10M - у меня ничего не нашлось, но команда сработала  
    Попробуем найти слово "error" в файлах директории /var/log:  
    grep -r "error" /var/log -выведет все совпадения в файлах директории /var/log  

### Разграничение прав доступа  

1. Создадим пользователя developer:  
    adduser developer - добавляем обычного пользователя  

2. Назначим права на опеределенную директорию /opt/devops-app:  
    chown -R doveloper /opt/devops-app - "chown -R" команда для изменения владельца директории и всех ее компонентов на указанного пользователя  

3. Запретим доступ к /backups пользователю developer:  
    chmod 770 /backups - "chmod" команда для изменения прав пользователей на файлы и директории, числа обозначают права доступа в данном случае, владелец и группа могут читать все, остальные пользователи нет  
    ls -ld /opt/devops-app - смотрим права на папку, она выводит тип, владельца, группу и права доступа (мой вывод: drwxrwxrwx 2 developer root 4096 Mar 28 13:06 /opt/devops-app)  
    ls -ld /backups - также смотрим права на папку (мой вывод: drwxrwx--- 2 root root 4096 Mar 28 13:10 /backups)  
    groups developer - посмотрим в каких группах состоит пользователь (если не выдавали отдельно права, то пользователь будет в группе users)  

### Работа с процессами  

1. Запустим простой веб-сервер командой:  
    python3 -m http.server 8080 - если до этого не был скачан python, то делается это командой apt install python3  
2. Проверим работоспособность сервера с помощью команды:
    curl http://localhost:8080 - должен вывести список файлов в консоли
3. Посмотрим на процессы запущенные в системе:
    ps -ef - стандартная команда для просмотра процессов (например у меня сервер python запущен на PID 4459)
    top - интерактивный мониторинг системных ресурсов, можно посмотреть какие процессы сколько занимают места (можно сортировать сочетаниями клавишь shift+p/m/t)  
    Завершим процесс сервера, необходимо выбрать его переключаясь на стрелочки и нажать клавишу "k".  
    Снова запустим сервер python  
    htop - улучшенная версия top, со списком запущенных процессов в реальном времени, цветами и прокруткой. Позволяет манипулировать процессами без ввода PID.  
    Удобно смотреть ресурсы системы.  
    kill - используется для отправки сигналов процессам.  
    Узнаем список всех сигналов и их номера командой: kill -l  
    Нам нужен сигналл простого завершения, будем снова сносить сервер командой: kill -15 <PID>  
    Но это не гарантирует остановку процесса при перехвате или в случае блокировки, поэтому можно принудительно завершить процесс командой: kill -9 <PID>  

### Написание скриптов  

1. Напишем обычный скрипт на bash, который будет:  
    a) Показывать uptime  
    b) Выводить 5 первых процессов системы  
    c) Использование диска системы  
    d) Открытые порты  
2. Создадим файл для bash-скрипта по адресу /scripts/system_report.sh:  
    touch /scripts/system_report.sh - "touch" стандартная команда для создания файлов  
    nano /scripts/system_report.sh - с помощью "nano" зайдем внутрь файла и начнем писать скрипт  
3. Написание скрипта на bash:
    Любой скрипт на bash начинается с шебанг и пути к bash: #!/bin/bash - указывает системе, что сценарий создан для bash  
    Далее можно вводить любые команды для выполнения, у меня будут команды для перечисленных выше целей:  
    uptime - показывает время неприрывной работы системы с момента последнего включения  
    ps aux | head -n 5 - показывает первые 5 процессов запущенные в системе с полной информацией о пользователе процесса  
    df -h - команда расшифровывается как disk free, за счет флага -h показывает размер в гигабайтах и мегабайтах  
    ss -tuln - команда для просмотра портов TCP (-t) и UDP (-u) в режиме прослушивания (-l) с числовыми значениями (-n)  
4. Установка разрешений для файла сценария  
    Сделаем файл исполняемым:  
    chmod +x ./scritp_reports.sh - ранее мы уже использовали эту команду для назначения прав пользователю на папку, теперь с помощью флага +x мы делаем файл исполняемым  
    cd /scripts | ./system_reports.sh - переходим в каталог /scripts и выполняем скрипт через команду ./system_reports.sh
